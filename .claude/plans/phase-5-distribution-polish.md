# Feature: Phase 5 — Distribution, CI/CD & Documentation

> Generated by `/plan-feature` — read CLAUDE.md and referenced files before implementing.

## Overview

Make PICE CLI installable, documented, and open-source ready. This phase adds GitHub Actions CI/CD, NPM binary distribution packages, a README, contributing guide, methodology docs, and telemetry HTTP sending. No new CLI commands — this is infrastructure, packaging, and documentation only.

## User Story

As a developer, I want to install PICE via `npm install -g pice` or `cargo install pice` and have a working CLI with documentation, so that I can start using structured AI coding workflows immediately.

## Notes

### Design decisions

- **reqwest with rustls-tls**: Avoids OpenSSL dependency which complicates cross-compilation (especially Linux ARM64). `rustls` is pure-Rust TLS.
- **Telemetry flush is best-effort**: We don't retry or queue failures beyond what's already in the SQLite queue. If the endpoint is down, events stay queued and will be sent on next CLI invocation. This matches the PRD's "non-fatal" requirement.
- **NPM package uses optionalDependencies**: The platform-specific packages are `optionalDependencies` so npm only downloads the binary for the current platform. This is the same pattern as Biome, Turbo, and esbuild.
- **docs/ vs .claude/docs/**: The `docs/` directory is the public-facing documentation. `.claude/docs/` contains the internal framework files scaffolded by `pice init`. Content is migrated but originals are preserved.
- **No greenfield.md or wisc-context.md yet**: The PRD lists these in the `docs/guides/` structure, but no source content exists in `.claude/docs/`. These can be added post-MVP or as community contributions.

### Risks

- **Cross-compilation for Linux ARM64** may require `cross` tool or Docker-based builds — GitHub Actions `ubuntu-latest` doesn't have ARM64 Rust target installed by default
- **NPM publishing requires auth tokens** configured as GitHub Secrets — document this in the release workflow
- **crates.io publishing requires API token** — same, needs GitHub Secret setup
- **Release workflow is complex** and can't be fully tested without a real tag push — recommend a dry-run with `--dry-run` flags first

### Follow-up work (post-MVP)

- Telemetry dashboard (web visualization of anonymous aggregate data)
- `docs/guides/greenfield.md` and `docs/guides/wisc-context.md`
- Homebrew formula for macOS
- AUR package for Arch Linux
- Windows installer (MSI)
- Refine `total_loops` metric to use `loop_events` table for lifecycle-based counting
